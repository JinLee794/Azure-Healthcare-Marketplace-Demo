# .runs/ — Workflow Run Outputs

This directory stores **all outputs generated by PA workflow runs**. Each run
gets a timestamped subdirectory with a clear, consistent structure.

> **This directory is gitignored.** Only this README is tracked.

## Directory Layout

```
.runs/
├── README.md                           ← this file (tracked)
├── latest → 2025-01-15T1430_PA-001A/   ← symlink to most recent run
├── 2025-01-15T1430_PA-001A/
│   ├── waypoints/
│   │   ├── assessment.json             ← Subskill 1 output (bead checkpoints)
│   │   └── decision.json              ← Subskill 2 output (human decision)
│   ├── outputs/
│   │   ├── audit_justification.md     ← Detailed audit document
│   │   ├── assessment_report.md       ← Formatted assessment report
│   │   ├── determination.json         ← Structured determination for downstream
│   │   ├── approval_letter.md         ← (if APPROVED)
│   │   ├── pend_letter.md            ← (if PENDING)
│   │   └── denial_letter.md          ← (if DENIED)
│   └── eval/
│       └── eval_results.json          ← Eval framework scores (if run)
└── 2025-01-16T0900_PA-002B/
    └── ...
```

## Naming Convention

Directories are named: `<YYYY-MM-DDTHHMM>_<REQUEST-ID>`

- **Timestamp** — ISO 8601 date + time (minutes), no colons for filesystem safety
- **Request ID** — the PA request identifier (e.g., `PA-001A`, `PA-20250115-3F2A1`)

## Finding Results

- **Latest run**: follow the `latest` symlink or sort alphabetically (newest last)
- **Specific case**: search by request ID in directory names
- **All assessments**: `find .runs -name assessment.json`
- **All outputs**: `find .runs -name "*.md" -path "*/outputs/*"`

## Lifecycle

- Created automatically by each `prior-auth` workflow run
- Not version-controlled (gitignored)
- Archive or delete old runs as needed
- `latest` symlink updated after each successful run
